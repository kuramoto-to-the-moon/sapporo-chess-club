# 札幌チェスクラブサイト 運用ガイド

非エンジニア向けの日常更新手順をまとめています。

すべての編集は **[Pages CMS](https://app.pagescms.org/kuramoto-to-the-moon/sapporo-chess-club/main)** から行います。GitHub アカウントとリポジトリへの招待が必要です。

---

## 編集できるもの一覧

| Pages CMS のメニュー | 内容 |
|---|---|
| **スケジュール（例会）** | 月 2 回の定期例会の日程 |
| **スケジュール（大会）** | 今後開催する大会の日程と申込フォーム設定 |
| **大会記録** | 終了した大会の記録 (要項 / 結果 / 棋譜) |
| **チェス講座** | サイト下部に表示される講座案内 |
| **外部リンク** | 「リンク / 資料」セクションのリンク (掲示板など) |
| **サイト基本情報** | メールアドレス、電話番号、会場、参加費 |

---

## 日常タスク別の手順

### A. 例会日程を追加する

1. Pages CMS で **「スケジュール（例会）」** を開く
2. 「例会日程」配列の末尾に新しい項目を追加
3. 各フィールドを入力:
   - **日付**: カレンダーから選択
   - **開始時間 / 終了時間**: HH:MM 形式 (例 09:00 / 15:00)
   - **部屋番号**: 例 810A、740 など
   - **会場**: 「かでる2.7」/「Kaderu 2.7」 (毎回同じ)
   - **備考**: 通常室と違う部屋を使う日などに「注意：810Aではなく740」のように入力 (任意)
4. 「保存」を押す → 自動で本番反映 (1〜2 分後)

> 💡 曜日は日付から自動計算されるので入力不要です。

---

### B. 大会の日程を追加する

複数日大会 (例: 土曜+日曜の 2 日連続) は **1 日 1 エントリ** 作成します。

1. Pages CMS で **「スケジュール（大会）」** を開く
2. 「大会日程」配列に大会の各日を追加
3. 各日のフィールド:
   - **開催日 / 開始時間 / 終了時間 / 部屋番号 / 会場**: 例会と同じ
   - **大会名**: 表示用 (例「札幌サマーチェス大会」/「Sapporo Summer Tournament」)
4. 申込フォームを出したい場合 (**初日にのみ設定**):
   - **Formspree ID**: Formspree 管理画面でフォームを作って取得した ID
   - **申込開始日 (任意)**: 受付開始日。未指定なら即時開始
   - **申込締切日 (任意)**: 未指定なら **開催日の前々日 23:59 が自動で締切**
5. 保存

> ⚠️ Formspree ID は **大会の初日のエントリにだけ** 設定してください。複数日に設定するとフォームが重複表示されます。

---

### C. 大会が終わったら結果を公開する

> 🚧 この手順は将来 [Issue #6](https://github.com/kuramoto-to-the-moon/sapporo-chess-club/issues/6) で半自動化予定。当面は手動です。

1. Pages CMS で **「スケジュール（大会）」** を開き、終わった大会のエントリを **削除**
2. 結果 PDF (`*.pdf`) と棋譜 PGN (`*NoAnalysis.pgn` / `*Analysis.pgn`) を準備
3. **「大会記録」** で「新規」 → 入力:
   - **大会名 (ja/en)**
   - **開催日**: 大会初日
   - **要項 PDF / 結果 PDF / 棋譜 PGN**: 「ファイルをアップロード」 → `competition` フォルダにアップロード
   - 「棋譜 PGN (解析付)」もあれば追加
4. 保存

> 💡 一覧ページでは **日付の新しい順** に自動で並びます。年やソート順を入力する必要はありません。

---

### D. 会場・連絡先・参加費を変更する

1. **「サイト基本情報」** を開く
2. 該当フィールド (メール / 電話 / 会場 / 参加費) を編集
3. 保存

---

### E. お知らせを追加する (将来機能、現状ブランチ隔離中)

`feature/announcements` ブランチで開発中。確定後にメインに統合予定。

機能概要:
- お知らせは 1 件 1 ファイルの markdown で管理 (Pages CMS から編集)
- 公開日が **直近 365 日以内** のお知らせは `/announcements/` (アクティブ) に表示
- **365 日経過すると自動で「アーカイブ」に移動** (`/announcements/archive/`)。編集者が手動で何かする必要はない
- アクティブページの末尾にアーカイブへのリンクが出る (アーカイブが空でない時のみ)
- ja で書けば en は GitHub Action が自動翻訳

---

### F. 講座情報を変更する

1. **「チェス講座」** から該当エントリを開く
2. タイトル / 説明文 / 外部 URL を編集
3. 外部 URL を設定すると講座セクションに「詳細を見る ↗」リンクが表示されます

---

### G. 外部リンクを追加する

1. **「外部リンク」** を開く
2. 「リンク一覧」配列に追加 (タイトル + URL)
3. 保存

> 日本チェス連盟 (JCA) のリンクは Footer に固定で表示されており、CMS からは編集しません。

## トラブルシューティング

### 保存したのにサイトに反映されない
- GitHub Actions のビルドが完了するまで 1〜3 分かかります
- 画面をハードリロード (Ctrl+Shift+R / Cmd+Shift+R)
- それでも出ない場合は管理者に連絡

### 入力した時刻フォーマットがエラーになる
- HH:MM (24 時間制) で入力してください。例: `09:00`、`14:30`
- `9:00` は NG → `09:00` で

### 過去の例会日程を消したい
- **削除しないでください**。過去の予定は表示されないだけで、データとしては残しておきます。誤って削除した場合は GitHub の履歴から復元できますが、管理者に連絡してください。

### 写真や画像をアップロードしたい
- 現在 CMS では大会関連 PDF / PGN ファイルしか扱えません。サイトに画像を追加したい場合は管理者に連絡してください。

### CMS にログインできない
- GitHub アカウントでログイン → リポジトリへの招待が必要です。管理者に依頼してください。

## 管理者向け補足

### Pages CMS のスキーマ
すべて `.pages.yml` で定義。`src/content.config.ts` (Astro 側スキーマ) と必ず一致させること。スキーマがずれると CMS の保存時にデータが落ちる事故が起きる。

### 自動翻訳 (将来機能)
お知らせ機能 (`feature/announcements` ブランチ) では Claude / DeepL API による自動 ja → en 翻訳を実装中。GitHub Secrets に `ANTHROPIC_API_KEY` または `DEEPL_API_KEY` を登録する必要あり。

### デプロイ
`main` への push で GitHub Actions が build → GitHub Pages にデプロイ。失敗時は Actions タブで確認。
